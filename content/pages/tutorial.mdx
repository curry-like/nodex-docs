# Tutorial

On this page, we get you up and running with UNiD's React Native SDK, embedding DID-wallet capabilities into your mobile/web applications. UNiD SDK allows you to reduce "honeypot" risks and improves user experience in privacy-preserving way.

If you don't already have an account and UNiD project established, head over to [unid.io](https://collabogate.com/products/index.html), then retune to this page.

- Install
- React Native SDK for mobile application
- Methods
	- DID Operations
	- Verifiable Credential
	- Authentication
- SDK for relying parties



## VC Data
This document contains the suggested specification for Verifiable Credential data, referred to as `crednetialSubject` property in the VC data model. In general, these schemas are based on [schema.org](https://schema.org/docs/full.html) base types such as [Place](https://schema.org/Place) and [Organization](https://schema.org/Organization), with custom extensions for new concepts not described by schema.org.
